# Решение ошибки подключения к серверу

## Ошибка: "Серверге туташуу болбой жатат"

Это означает, что мобильное приложение не может подключиться к бэкенду серверу.

### Решение зависит от того, где запущено приложение:

#### 1. **Android Эмулятор** 
- ✅ Автоматически использует `10.0.2.2:8001`
- Проверьте что сервер запущен: `curl http://localhost:8001/docs`

#### 2. **Physical Android Device**
- Нужно заменить URL на IP адрес вашего компьютера
- Найдите IP адрес: 
  ```bash
  ifconfig | grep "inet " | grep -v 127.0.0.1
  ```
  Результат: например `192.168.1.100`

- Отредактируйте файл: `frontend/lib/core/config.dart`
  ```dart
  if (defaultTargetPlatform == TargetPlatform.android) {
    return 'http://192.168.1.100:8001';  // Измените IP адрес
  }
  ```

- Убедитесь что сервер слушает на всех интерфейсах:
  ```bash
  uvicorn app.main:app --host 0.0.0.0 --port 8001
  ```

#### 3. **iOS Simulator**
- ✅ Может использовать `127.0.0.1:8001`
- Проверьте что сервер запущен

#### 4. **Physical iOS Device**
- Замените URL на IP адрес вашего компьютера (как для Android)
- Отредактируйте `frontend/lib/core/config.dart`:
  ```dart
  return 'http://192.168.1.100:8001';  // Ваш IP адрес
  ```

### Проверка подключения:

```bash
# Убедитесь что сервер работает
curl -X GET http://localhost:8001/docs

# Проверьте что сервер доступен с вашего IP адреса
curl -X GET http://192.168.1.100:8001/docs
```

### Еще один вариант - используй Environment Variable:

При сборке приложения укажите API URL:

```bash
flutter run --dart-define=API_BASE_URL=http://192.168.1.100:8001
```

### Отладка:

Если ошибка все еще происходит, приложение покажет URL где оно пытается подключиться. Используйте этот URL для проверки подключения вручную.
